# 人才評級系統 - 邊緣案例處理

## 📋 案例總覽

本文檔記錄人才評級系統在處理特殊情況時的處理邏輯與評分標準。

---

## 1️⃣ 社會新鮮人（0 年經驗）

### 定義
- 總工作年資 < 0.5 年
- 剛畢業或剛開始第一份工作

### 評分邏輯

| 維度 | 評分方式 | 說明 |
|------|---------|------|
| **學歷背景 (20%)** | 正常評分 | 學歷是主要評估項目 |
| **工作年資 (20%)** | 5 分 | 給予基礎分（避免 0 分） |
| **技能廣度 (20%)** | 正常評分 | 根據學校專案、實習、證照 |
| **工作穩定性 (20%)** | 10 分 | 無法評估，給予中間分 |
| **職涯發展軌跡 (10%)** | 7 分 | 無法評估，給予中間分 |
| **特殊加分 (10%)** | 正常評分 | 名校、實習、開源貢獻 |

### 評級範圍
- **最高**：S 級（92 分）- 名校博士 + 豐富技能 + 開源貢獻
- **一般**：B 級（60-69 分）- 學士學位 + 基礎技能
- **最低**：C 級（45 分）- 高中學歷 + 無技能資料

### 範例

**案例 A**：台大資工碩士應屆畢業生
```json
{
  "name": "張小明",
  "education": [{"degree": "碩士", "school": "國立台灣大學", "major": "資訊工程"}],
  "total_years": 0,
  "skills": ["Python", "Java", "React", "Node.js", "Docker", "AWS"],
  "stability": 50,
  "work_history": []
}
```

**評分結果**：
- 學歷：20/20（碩士 18 + 台大 2）
- 年資：5/20（社會新鮮人）
- 技能：15/20（6 個技能）
- 穩定性：10/20（無法評估）
- 軌跡：7/10（無法評估）
- 加分：3/10（名校）
- **總分**：60/100 → **B 級**

## 2️⃣ 只有一份工作（無法計算穩定性）

### 定義
- 工作經歷只有 1 筆
- 無離職記錄，無法計算「轉職次數」

### 評分邏輯

| 維度 | 評分方式 | 說明 |
|------|---------|------|
| **學歷背景 (20%)** | 正常評分 | - |
| **工作年資 (20%)** | 正常評分 | 根據該份工作的年資 |
| **技能廣度 (20%)** | 正常評分 | - |
| **工作穩定性 (20%)** | **特殊處理** | 見下方說明 |
| **職涯發展軌跡 (10%)** | 7 分 | 無法評估晉升，給中間分 |
| **特殊加分 (10%)** | 正常評分 | - |

### 穩定性評分規則

**情況 A**：在職中（current job）
- 任期 ≥ 3 年 → 18/20（穩定）
- 任期 1-3 年 → 15/20（普通）
- 任期 < 1 年 → 12/20（觀察中）

**情況 B**：已離職
- 任期 ≥ 5 年 → 15/20（長期穩定）
- 任期 2-5 年 → 12/20（普通）
- 任期 < 2 年 → 10/20（可能不穩定）

### 範例

**案例 B**：在同一家公司工作 4 年的資深工程師
```json
{
  "name": "李大華",
  "education": [{"degree": "學士", "school": "國立交通大學"}],
  "total_years": 4,
  "skills": ["Python", "Django", "PostgreSQL", "Redis", "AWS", "Docker", "Kubernetes"],
  "stability": 85,
  "work_history": [
    {
      "company": "某科技公司",
      "position": "Senior Backend Engineer",
      "duration": 48,
      "current": true
    }
  ]
}
```

**評分結果**：
- 學歷：17/20（學士 15 + 交大 2）
- 年資：15/20（4 年）
- 技能：18/20（7 個技能）
- 穩定性：18/20（在職 ≥3 年）
- 軌跡：7/10（單一工作）
- 加分：2/10（名校）
- **總分**：77/100 → **A 級**

## 3️⃣ 職涯空窗期（Gap Year）

### 定義
- 工作經歷間有 >6 個月的空白期
- 可能原因：進修、旅行、照顧家人、創業失敗

### 評分邏輯

**不扣分情況**：
- 空窗期用於進修（碩士、博士、證照）
- 空窗期 <1 年

**扣分情況**：
- 空窗期 1-2 年且無合理說明 → 穩定性 -2 分
- 空窗期 >2 年且無合理說明 → 穩定性 -5 分

### 判斷方式
1. 檢查 `work_history` 的時間軸
2. 比對 `education` 是否有對應時間的進修記錄
3. 檢查備註欄是否有說明

### 範例

**案例 C**：中間有 1 年空窗期
```json
{
  "work_history": [
    {"company": "A 公司", "start": "2018-01", "end": "2020-06"},
    {"company": "B 公司", "start": "2021-07", "end": "2024-02"}
  ],
  "education": [
    {"degree": "碩士", "school": "某大學", "start": "2020-09", "end": "2021-06"}
  ]
}
```

**評分**：空窗期用於碩士進修 → **不扣分**

## 4️⃣ 頻繁轉職（Job Hopper）

### 定義
- 3 年內轉職 ≥3 次
- 平均任期 <1 年

### 評分邏輯

**穩定性評分**：
- 轉職 3 次/3 年 → 穩定性 50 → 12/20
- 轉職 4 次/3 年 → 穩定性 40 → 10/20
- 轉職 5+ 次/3 年 → 穩定性 30 → 8/20

**但考慮特殊情況**：
- ✅ 每次轉職都是晉升（職位等級提升）→ 軌跡加分
- ✅ 新創公司倒閉（非個人因素）→ 穩定性不扣分
- ✅ 合約/派遣工作（短期性質）→ 穩定性不扣分

### 範例

**案例 D**：快速晉升的 Job Hopper
```json
{
  "total_years": 5,
  "work_history": [
    {"position": "Junior Engineer", "duration": 12},
    {"position": "Engineer", "duration": 18},
    {"position": "Senior Engineer", "duration": 14},
    {"position": "Lead Engineer", "duration": 16}
  ],
  "stability": 65
}
```

**評分**：
- 穩定性：15/20（雖然轉職 4 次，但每次都晉升）
- 軌跡：10/10（持續晉升）
- **總分提升**：軌跡加分彌補穩定性扣分

## 5️⃣ 自由工作者 / 接案者

### 定義
- 工作經歷標註為「Freelancer」「自由工作者」
- 有多個短期專案而非固定雇主

### 評分邏輯

**年資計算**：
- 累計所有專案時間（可能重疊）
- 取最早開始到最晚結束的總時長

**穩定性評分**：
- 持續接案 >3 年 → 15/20（穩定自由工作者）
- 接案 1-3 年 → 12/20（轉型期）
- 接案 <1 年 → 10/20（觀察中）

**技能評分**：
- 自由工作者通常技能廣度較高 → 正常評分
- 加分：如有知名客戶（Google, Meta 等）

### 範例

**案例 E**：資深自由工作者
```json
{
  "total_years": 6,
  "work_history": [
    {"company": "Freelance", "position": "Full Stack Developer", "duration": 72}
  ],
  "skills": ["React", "Node.js", "Python", "AWS", "Docker", "MongoDB", "PostgreSQL", "GraphQL"],
  "stability": 75
}
```

**評分**：
- 穩定性：15/20（持續接案 6 年）
- 技能：18/20（8 個技能）
- **總分**：與一般員工相當或更高

## 6️⃣ 跨領域轉職

### 定義
- 職位類型完全改變（如：工程師 → PM、業務 → 工程師）
- 產業完全改變（如：金融 → 科技、製造 → 服務）

### 評分邏輯

**年資計算**：
- 累計總年資（不區分領域）
- 新領域年資單獨標註

**技能評分**：
- 評估「可轉移技能」（Transferable Skills）
- 跨領域技能（如 PM 轉工程師，專案管理能力仍有價值）

**軌跡評分**：
- 合理轉職（有學習準備期）→ 7/10
- 向上轉職（薪資/職級提升）→ 8/10
- 向下轉職（降級重新開始）→ 5/10

### 範例

**案例 F**：業務轉工程師
```json
{
  "total_years": 8,
  "work_history": [
    {"position": "Sales Manager", "duration": 60, "industry": "Software"},
    {"position": "Junior Developer", "duration": 36, "industry": "Software"}
  ],
  "skills": ["Python", "JavaScript", "SQL", "Sales", "Project Management"],
  "education": [{"degree": "資訊工程學士", "type": "在職進修"}]
}
```

**評分**：
- 年資：18/20（8 年總年資）
- 技能：15/20（5 個技能，技術 + 業務）
- 軌跡：7/10（合理轉職，有在職進修）
- 加分：+2（跨領域經驗豐富）
- **總分**：可達 A 級

## 7️⃣ 無工作經歷但有作品集

### 定義
- 工作年資 0
- 但有 GitHub 開源專案、個人作品、競賽得獎

### 評分邏輯

**年資**：5/20（基礎分）

**技能**：根據作品集評估
- GitHub stars >100 → 視同 1 年經驗
- 完整作品集（3+ 專案）→ 技能 +5 分

**加分**：
- GitHub 開源貢獻 +2
- 競賽得獎 +2
- 技術社群講者 +1

### 範例

**案例 G**：開源貢獻者
```json
{
  "total_years": 0,
  "github_url": "https://github.com/awesome-dev",
  "skills": ["TypeScript", "React", "Node.js", "Docker", "Kubernetes", "PostgreSQL"],
  "projects": [
    {"name": "某熱門框架", "stars": 500},
    {"name": "某工具套件", "stars": 200}
  ]
}
```

**評分**：
- 年資：5/20（基礎）
- 技能：20/20（6 個技能 + 開源專案）
- 加分：10/10（開源 +2, 高 stars +5, 社群 +3）
- **總分**：可達 A+ 級
